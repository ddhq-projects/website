# RFC: MVP Одностраничник ddhq.ru

**Статус:** Draft
**Автор:** DDHQ team
**Дата:** 2026-06-11

---

## 1. Введение

У Delta Development Headquarters (DDHQ) нет публичного сайта. Есть заглушка в репозитории `ddhq-site` на Astro (болванка с текстом «website»), есть дизайн-система (`DESIGN.md`) и маркетинговая стратегия (`MARKETING.md`), но живой страницы — нет.

Этот RFC описывает создание одностраничного сайта-визитки. Один экран прокрутки, который за 10 секунд отвечает на вопросы: кто, что делают, на чём и как связаться.

Цель — **запустить MVP как можно быстрее** (на этой неделе), а итерации делать уже поверх работающего сайта.

---

## 2. Цели и причины

### Цели
- Дать команде публичную точку входа: `ddhq.ru`
- Объяснить за 10 секунд, чем занимается DDHQ: «делаем сайты, веб-приложения, сервисы и чат-ботов — просто и понятно»
- Дать способ связаться (CTA: email `m@ddhq.site`)
- Заложить основу для последующих итераций (добавление проектов, блога, страниц продуктов)

### Причины / Мотивация
- Сейчас сайта нет — команда невидима в поиске и при личных рекомендациях
- Заглушка в репозитории существует, но не отражает суть
- Дизайн-система и маркетинговый фреймворк уже готовы — нужна только вёрстка
- Сайт нужен как якорь для всех остальных каналов (GitHub, Telegram, статьи)

## 3. Составляющие и особенности

### 3.1. Структура страницы (5 секций)

Одностраничник с вертикальным скроллом. Порядок секций:

#### 3.1.1. Hero
- Заголовок: «Делаем веб понятным и простым» (или вариант)
- Подзаголовок: одна фраза о том, что именно делаем — сайты, веб-приложения, сервисы, чат-боты
- CTA-кнопка: «Написать нам» → `mailto:m@ddhq.site`
- Визуально: минималистично, тёмный или светлый фон с акцентным градиентом

#### 3.1.2. Что делаем
- 4 карточки (или ряда) с направлениями:
  - **Веб-сайты** — лендинги, визитки, порталы
  - **Веб-приложения** — админки, дашборды, SaaS
  - **Сервисы и автоматизация** — интеграции, боты, пайплайны
  - **AI и чат-боты** — агенты, LLM-интеграции
- Каждая карточка: иконка + название + одна строка описания

#### 3.1.3. Стек
- Технологический стек, на котором работает команда
- Оформлен визуально: бейджи или иконки технологий (Astro, Next.js, Laravel, PHP, WordPress, TypeScript, Tailwind, Docker и т.д.)
- Без подробных описаний — узнаваемые логотипы и названия

#### 3.1.4. Проекты
- Сетка из 2-4 карточек проектов
- Каждая: название, одна строка описания, ссылка (GitHub, отдельный сайт или заглушка «coming soon»)
- На старте можно показать 1-2 реальных проекта + 1-2 «in progress»

#### 3.1.5. Контакты
- Призыв к действию: «Напишите нам — обсудим задачу»
- Единственный контакт: email `m@ddhq.site`
- Никаких форм — просто `mailto:` ссылка

### 3.2. Навигация

- Фиксированная верхняя панель (sticky header) с названием «DDHQ» и якорными ссылками на секции
- На мобильных — бургер-меню или компактный sticky header только с логотипом
- Плавный скролл к секциям

### 3.3. Технические решения

- **Фреймворк:** Astro 6.x (уже в проекте)
- **Стили:** Tailwind CSS 3.4 + DaisyUI 4.12 (уже в проекте)
- **Структура:** вся страница — один `.astro` файл (`src/pages/index.astro`), секции как компоненты в `src/components/`
- **Изображения:** SVG-иконки для стека, оптимизированные скриншоты для проектов (если есть)
- **Мобильная адаптация:** DaisyUI responsive-утилиты, mobile-first где возможно
- **Производительность:** Astro статический вывод, минимум JS, lighthouse > 90

### 3.4. Архитектурные решения (ADR)

- **Решение:** Использовать DaisyUI-компоненты вместо кастомного CSS. **Почему:** ускоряет разработку, поддерживает единообразие, соответствует DESIGN.md
- **Решение:** Все секции — отдельные Astro-компоненты в `src/components/`. **Почему:** поддерживаемость, возможность переиспользовать на других страницах в будущем
- **Решение:** Контакты — только `mailto:m@ddhq.site`, без форм. **Почему:** ускоряет запуск, нет зависимости от бэкенда, проще некуда
- **Отвергнуто:** Многостраничный сайт сразу. **Почему:** дольше, а MVP нужен ASAP. Одностраничник покрывает все потребности старта.

## 4. Критерии готовности

Definition of Done — конкретные, проверяемые условия:

- [x] Все 5 секций свёрстаны и отображаются на десктопе и мобильных (responsive)
- [x] Sticky header с якорной навигацией работает
- [x] Цвета, шрифты, отступы соответствуют DESIGN.md
- [x] Тексты соответствуют правилам MARKETING.md (без buzzwords, без канцелярита)
- [x] `pnpm build` проходит без ошибок
- [x] Сайт открывается на `localhost` через `pnpm dev`
- [x] Проставлены базовые SEO-метатеги (title, description, og:image)
- [ ] Lighthouse: Performance ≥ 90, Accessibility ≥ 90

### Тестирование
- Визуальная проверка на десктопе (1440px) и мобильном (375px)
- Проверка якорных ссылок — все ведут к нужным секциям
- Проверка внешних ссылок (Telegram, email, GitHub)
- Скриншотный тест для будущего сравнения (если настроен)

### Мониторинг / Метрики
- После деплоя: подключить базовую аналитику (яндекс.метрика или аналог)
- Отслеживать: кол-во визитов, клики по CTA, время на странице
- Эти метрики — база для итераций и A/B-тестов

- Проект уже настроен (Astro, Tailwind, DaisyUI) — блокеров нет

| Дизайн «не смотрится» | Низкая | Следовать DESIGN.md, использовать DaisyUI-компоненты |
